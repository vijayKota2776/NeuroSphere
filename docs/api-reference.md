## Common Patterns

### Base URLs

| Environment | Base URL |
|-------------|----------|
| Local (direct) | `http://localhost:{service_port}` |
| Local (gateway) | `http://localhost:8080` |
| **Live EC2** | **`http://13.126.102.15:{service_port}`** |
| **Live EC2 (gateway)** | **`http://13.126.102.15:8080`** |
| Staging | `https://api-staging.neurosphere.io` |
| Production | `https://api.neurosphere.io` |

### Standard Endpoints (All Services)

Every NeuroSphere service exposes these three endpoints:

| Method | Path | Description | Response |
|--------|------|-------------|----------|
| `GET` | `/health` | Liveness probe | `{"status": "healthy", "service": "<name>"}` |
| `GET` | `/ready` | Readiness probe | `{"status": "ready", ...}` or `503` |
| `GET` | `/metrics` | Prometheus metrics | Prometheus text format |

### Response Format

All API responses use JSON with consistent structure:

```json
{
  "field": "value",
  "timestamp": "2026-06-20T01:28:53.000Z"
}
```

### Error Responses

```json
{
  "error": "Description of what went wrong"
}
```

| HTTP Code | Meaning |
|-----------|---------|
| `200` | Success |
| `201` | Created |
| `202` | Accepted (queued for processing) |
| `400` | Bad Request (validation error) |
| `404` | Not Found |
| `409` | Conflict (command rejected due to state) |
| `429` | Rate Limited |
| `500` | Internal Server Error |
| `502` | Service Unavailable (upstream down) |
| `503` | Service Not Ready |

### Request Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Content-Type` | Yes (POST) | `application/json` |
| `X-Request-ID` | No | Auto-generated by gateway if absent |
| `X-Surgeon-ID` | No | Operator identification for audit |

### POST /api/robots/command

Send a command to a surgical robot.

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `robot_id` | string | Yes | Target robot identifier |
| `command` | string | Yes | One of: `move`, `calibrate`, `start_procedure`, `stop_procedure`, `emergency_halt` |
| `parameters` | object | No | Command-specific parameters |

**Command Parameters:**

| Command | Parameter | Type | Description |
|---------|-----------|------|-------------|
| `move` | `target_position` | string | Target position name (e.g., "home", "park") |
| `calibrate` | — | — | No parameters needed |
| `start_procedure` | `procedure` | string | Procedure name (e.g., "laparoscopic_cholecystectomy") |
| `stop_procedure` | — | — | No parameters needed |
| `emergency_halt` | — | — | No parameters needed |

**Example — Start a procedure:**
```bash
curl -s -X POST http://localhost:5050/api/robots/command \
  -H "Content-Type: application/json" \
  -d '{
    "robot_id": "NSR-DA-VINCI-001",
    "command": "start_procedure",
    "parameters": {
      "procedure": "laparoscopic_cholecystectomy"
    }
  }' | jq .
```

**Response (200 — Success):**
```json
{
  "command_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "robot_id": "NSR-DA-VINCI-001",
  "command": "start_procedure",
  "status": "success",
  "detail": "Procedure 'laparoscopic_cholecystectomy' started",
  "latency_ms": 127.3,
  "timestamp": "2026-06-20T01:28:53.000Z"
}
```

**Response (409 — Rejected):**
```json
{
  "command_id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "robot_id": "NSR-DA-VINCI-001",
  "command": "start_procedure",
  "status": "rejected",
  "detail": "Robot already performing 'prostatectomy'",
  "latency_ms": 89.5,
  "timestamp": "2026-06-20T01:28:53.000Z"
}
```

**Example — Emergency halt:**
```bash
curl -s -X POST http://localhost:5050/api/robots/command \
  -H "Content-Type: application/json" \
  -d '{
    "robot_id": "NSR-DA-VINCI-001",
    "command": "emergency_halt"
  }' | jq .
```

**Example — Move robot:**
```bash
curl -s -X POST http://localhost:5050/api/robots/command \
  -H "Content-Type: application/json" \
  -d '{
    "robot_id": "NSR-ROSA-001",
    "command": "move",
    "parameters": {"target_position": "home"}
  }' | jq .
```

**Response (400):**
```json
{
  "error": "robot_id and command are required"
}
```

```json
{
  "error": "Invalid command. Must be one of ['move', 'calibrate', 'start_procedure', 'stop_procedure', 'emergency_halt']"
}
```

## 2. Diagnostic Engine Service (Port 3000)

Base URL: `http://localhost:3000` · **Live:** `http://13.126.102.15:3000`

### POST /api/diagnostics/analyze

Submit a diagnostic scan for AI analysis. The job is queued and processed asynchronously.

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `patient_id` | string | Yes | Patient identifier |
| `scan_type` | string | Yes | One of: `ct_scan`, `mri`, `xray`, `ultrasound`, `pet_scan`, `mammogram` |
| `body_region` | string | No | One of: `head`, `chest`, `abdomen`, `spine`, `pelvis`, `extremities` (default: `chest`) |
| `priority` | string | No | One of: `stat`, `urgent`, `routine` (default: `routine`) |

**Request:**
```bash
curl -s -X POST http://localhost:3000/api/diagnostics/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "patient_id": "PAT-001",
    "scan_type": "ct_scan",
    "body_region": "chest",
    "priority": "stat"
  }' | jq .
```

**Response (202 — Accepted):**
```json
{
  "message": "Diagnostic analysis queued successfully",
  "job_id": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "estimated_time_seconds": 45,
  "priority": "stat",
  "queue_position": 3
}
```

**Response (400):**
```json
{
  "error": "scan_type must be one of: ct_scan, mri, xray, ultrasound, pet_scan, mammogram"
}
```

## 4. Telemetry Ingest Service (Port 5002)

Base URL: `http://localhost:5002` · **Live:** `http://13.126.102.15:5002`

### POST /api/telemetry/ingest

Ingest one or more telemetry events. Accepts a single event object or an array of events.

**Event Schema:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source_id` | string | Yes | Source device identifier (e.g., `NSR-DA-VINCI-001`) |
| `source_type` | string | Yes | One of: `surgical_robot`, `patient_monitor`, `env_sensor`, `gateway` |
| `event_type` | string | Yes | One of: `heartbeat`, `position_update`, `error`, `vital_sign`, `calibration` |
| `timestamp` | string | Yes | ISO 8601 timestamp |
| `payload` | object | Yes | Event-specific data |

**Request — Single event:**
```bash
curl -s -X POST http://localhost:5002/api/telemetry/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "source_id": "NSR-DA-VINCI-001",
    "source_type": "surgical_robot",
    "event_type": "heartbeat",
    "timestamp": "2026-06-20T01:28:53.000Z",
    "payload": {
      "battery_level": 87.3,
      "cpu_temperature": 42.1,
      "memory_usage_pct": 34.5,
      "joint_positions": [0.0, 15.2, -3.1, 90.0, 0.5, -12.3]
    }
  }' | jq .
```

**Request — Multiple events:**
```bash
curl -s -X POST http://localhost:5002/api/telemetry/ingest \
  -H "Content-Type: application/json" \
  -d '[
    {
      "source_id": "NSR-DA-VINCI-001",
      "source_type": "surgical_robot",
      "event_type": "position_update",
      "timestamp": "2026-06-20T01:28:53.000Z",
      "payload": {"x": 12.5, "y": -3.2, "z": 45.1}
    },
    {
      "source_id": "ENV-SENSOR-OR3",
      "source_type": "env_sensor",
      "event_type": "vital_sign",
      "timestamp": "2026-06-20T01:28:53.000Z",
      "payload": {"temperature": 21.3, "humidity": 45.2}
    }
  ]' | jq .
```

**Response (202):**
```json
{
  "accepted": 2,
  "rejected": 0,
  "errors": null
}
```

**Response with validation errors (202):**
```json
{
  "accepted": 1,
  "rejected": 1,
  "errors": [
    "Event 1: missing required field 'source_type'"
  ]
}
```

### POST /api/telemetry/ingest/batch

Batch ingestion optimized for high-throughput producers. Accepts up to 1,000 events per request.

**Request:**
```bash
curl -s -X POST http://localhost:5002/api/telemetry/ingest/batch \
  -H "Content-Type: application/json" \
```

## 5. API Gateway (Port 8080)

Base URL: `http://localhost:8080` · **Live:** `http://13.126.102.15:8080`

All upstream service endpoints are accessible through the gateway with the same paths. The gateway adds:

### Response Headers

Every response from the gateway includes:

| Header | Value | Purpose |
|--------|-------|---------|
| `X-Request-ID` | UUID | Distributed tracing correlation |
| `X-Content-Type-Options` | `nosniff` | MIME type sniffing prevention |
| `X-Frame-Options` | `DENY` | Clickjacking protection |
| `X-XSS-Protection` | `1; mode=block` | XSS filter |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | Referrer control |
| `Access-Control-Allow-Origin` | `*` | CORS |
| `Access-Control-Allow-Methods` | `GET, POST, PUT, DELETE, PATCH, OPTIONS` | CORS methods |

### Routing Table

```bash
# All these work through the gateway at port 8080:

# Robot Command
curl http://localhost:8080/api/robots/status
curl http://localhost:8080/api/robots/heartbeat
curl http://localhost:8080/api/robots/NSR-DA-VINCI-001
curl -X POST http://localhost:8080/api/robots/command -H "Content-Type: application/json" -d '...'

# Diagnostic Engine
curl -X POST http://localhost:8080/api/diagnostics/analyze -H "Content-Type: application/json" -d '...'
curl http://localhost:8080/api/diagnostics/queue
curl http://localhost:8080/api/diagnostics/results/{jobId}
curl http://localhost:8080/api/diagnostics/stats

# Patient Monitor
curl http://localhost:8080/api/patients/vitals
curl http://localhost:8080/api/patients/vitals/{patient_id}
curl http://localhost:8080/api/patients/alerts
curl http://localhost:8080/api/patients/dashboard
curl -X POST http://localhost:8080/api/patients/register -H "Content-Type: application/json" -d '...'
curl http://localhost:8080/api/patients/history/{patient_id}

# Telemetry Ingest
curl -X POST http://localhost:8080/api/telemetry/ingest -H "Content-Type: application/json" -d '...'
curl -X POST http://localhost:8080/api/telemetry/ingest/batch -H "Content-Type: application/json" -d '...'
curl http://localhost:8080/api/telemetry/stats
curl http://localhost:8080/api/telemetry/recent
curl http://localhost:8080/api/telemetry/errors
curl http://localhost:8080/api/telemetry/health-summary

# Gateway Health
curl http://localhost:8080/health
```

### Error Responses

The gateway returns custom JSON error responses:

**429 — Rate Limited:**
```json
{
  "error": "rate_limit_exceeded",
  "message": "Request rate limit exceeded. Medical systems enforce strict rate limits for patient safety. Please retry after a brief pause.",
  "code": 429
}
```

**502 — Bad Gateway:**
```json
{
  "error": "service_unavailable",
  "message": "The requested medical service is temporarily unreachable. Failover protocols are active. Clinical staff: please use backup communication channels if this persists.",
  "code": 502
}
```

**503 — Service Maintenance:**
```json
{
  "error": "service_maintenance",
  "message": "This medical robotics service is undergoing scheduled maintenance. All active surgical procedures have been safely transitioned to manual control. Estimated recovery: <5 minutes.",
  "code": 503
}
```

**504 — Gateway Timeout:**
```json
{
  "error": "gateway_timeout",
  "message": "The upstream medical service did not respond within the safety timeout window. Automated watchdog has been notified. No patient procedures are affected.",
  "code": 504
}
```

**404 — Not Found:**
```json
{
  "error": "not_found",
  "message": "The requested endpoint does not exist on the NeuroSphere Gateway. Available API prefixes: /api/robots, /api/diagnostics, /api/patients, /api/telemetry",
  "code": 404
}
```
